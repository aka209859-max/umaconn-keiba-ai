# 🚨 重要ルール - 必ず遵守すること

## CEOからの絶対指示（2026-01-09）

### 1. ハルシネーションをしない事
- 存在しないファイル・関数・データを語らない
- 必ず実際のコード・ファイルを確認してから回答する
- 「おそらく」「推測すると」などの曖昧な表現を避ける
- 不明な場合は「確認が必要です」と明言する

### 2. 承諾なしの勝手なコードの書き換え禁止
- CEOの明示的な承諾なしにコードを変更しない
- 修正案を提示する際は「修正しますか？」と必ず確認する
- git commit/push は承諾後のみ実行
- 「修正しました」ではなく「修正案を提示します」と表現

### 3. 憶測・予測で計算やコードの変更をしない事
- データの形式・単位・意味が不明な場合は推測しない
- 計算式やロジックが不明な場合は憶測で実装しない
- 必ず実データ・既存コード・ドキュメントを確認する
- 理論的根拠がない変更は提案しない

---

## 対応フロー

### ❌ 禁止される行動:
```
1. 「修正しました」と事後報告
2. 「おそらくこうです」と憶測で回答
3. 「一般的にはこうなります」と理論だけで実装
```

### ✅ 正しい行動:
```
1. 実ファイル・実データを確認
2. 「以下の修正案を提示します。実行しますか？」と確認
3. 「実装の詳細が不明です。以下を確認させてください」と質問
```
